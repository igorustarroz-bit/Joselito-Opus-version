# UI02-Button

Botón principal del sistema. Según Figma hay **tres tipos** con usos distintos
(Primary/Secondary/Terciary), tamaños **XS–L** y estados hover/focus/selected/disabled.
Es **rectangular** (sin radio) y el texto va en **mayúsculas** (estilo CTA, 14/18, tracking 0.5px).
Admite iconos opcionales a izquierda y/o derecha.

## Anatomía

1. **Contenedor** (`.jl-btn`): caja rectangular en `inline-flex`, centrada, con borde de 1px y espaciado interno según el tamaño.
2. **Icono izquierdo** (`.jl-btn__icon`, opcional): se muestra cuando se pasa `iconLeft`.
3. **Etiqueta** (`<span>`): texto del botón en mayúsculas, solo se renderiza si hay `children`.
4. **Icono derecho** (`.jl-btn__icon`, opcional): se muestra cuando se pasa `iconRight`.

El tamaño del icono se deriva del tamaño del botón: `xs`/`s` usan icono `XS`, el resto icono `S`.

## Subtemas

Hereda el subtema del contenedor a través de `data-theme`: los colores se resuelven mediante tokens
semánticos, y un hijo puede sobrescribir al padre estableciendo su propio subtema.
El componente usa su **grupo de color propio** `--btn-<tipo>-<estado>-<fill|text|stroke|icon>`
(grupo Button de Semantic-Color), que se adapta automáticamente al subtema activo.

## Comportamiento

- **Default**: colores base del tipo.
- **Hover** (`:hover:not(:disabled)`): cambia fill/text/stroke/icon a los tokens de hover.
- **Focus** (`:focus-visible`): estado de foco con `outline: none`, el foco visible se aporta vía tokens de color.
- **Selected** (`data-selected="true"`, prop `selected`): estado activo persistente.
- **Disabled** (`:disabled`, prop `disabled`): colores atenuados y `cursor: not-allowed`; se ignora el hover.
- Es un botón no controlado; el estado `selected` lo gobierna quien lo usa.
- El texto no hace wrap (`white-space: nowrap`); el ancho respeta un `min-width` por tamaño.

## Tokens

Colores por tipo y estado (grupo Button de Semantic-Color, adaptados por subtema):

| Token | Uso |
| --- | --- |
| `--btn-<tipo>-<estado>-fill` | Fondo (tipo ∈ primary/secondary/terciary; estado ∈ default/hover/focused/selected/disabled) |
| `--btn-<tipo>-<estado>-text` | Color de texto |
| `--btn-<tipo>-<estado>-stroke` | Color del borde |
| `--btn-<tipo>-<estado>-icon` | Color del icono |
| `--btn-<size>-size` | Altura del botón (size ∈ xs/s/m/l/xl) |
| `--btn-<size>-padding` | Padding lateral |
| `--btn-<size>-min-width` | Ancho mínimo |
| `--sp-2` | Separación entre icono y texto (8px) |
| `--font-family-cta`, `--font-weight-cta`, `--letter-spacing-cta` | Tipografía CTA |

## Propiedades

| Prop | Tipo | Default | Descripción |
| --- | --- | --- | --- |
| `type` | `primary \| secondary \| terciary` | `primary` | Tipo de botón |
| `size` | `xs \| s \| m \| l` | `m` | Tamaño |
| `selected` | `boolean` | `false` | Estado seleccionado |
| `disabled` | `boolean` | `false` | Deshabilitado |
| `iconLeft` / `iconRight` | `string` | — | Nombre de icono (ver Icon) |
| `children` | `node` | — | Texto del botón |
| `className` | `string` | `''` | Clases adicionales |

## Accesibilidad

- Renderiza un `<button type="button">` nativo, con foco y activación por teclado por defecto.
- Estado `disabled` mediante el atributo nativo `disabled`.
- El foco visible se apoya en los tokens de color del estado `focused`; el contraste se garantiza vía tokens semánticos adaptados al subtema.
- Si el botón se usa solo con icono (sin texto), considera aportar una etiqueta accesible o usar **UI03-Button-Icon**, que gestiona `aria-label`.
