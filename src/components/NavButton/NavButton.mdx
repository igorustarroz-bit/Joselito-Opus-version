# UI01-Nav-Button

Botón de **navegación** (uno de los tres tipos de botón del sistema). Tamaño fijo XS
(alto 40, padding 16), texto CTA-XS en mayúsculas (12/14, tracking 0.5px), rectangular.
Por defecto muestra un **caret** a la derecha (indica menú desplegable).

## Anatomía

1. **Contenedor** (`.jl-navbtn`): caja rectangular en `inline-flex`, centrada, con borde de 1px, alto fijo XS y padding lateral fijo.
2. **Icono izquierdo** (`.jl-navbtn__icon`, opcional): se muestra cuando se pasa `iconLeft`.
3. **Etiqueta** (`<span>`): texto en mayúsculas, solo si hay `children`.
4. **Icono derecho** (`.jl-navbtn__icon`): por defecto un `CaretDown` que indica desplegable; se puede sustituir o anular con `iconRight`.

Los iconos usan tamaño `XS`.

## Subtemas

Hereda el subtema del contenedor a través de `data-theme`; los colores se resuelven mediante tokens
semánticos y un hijo puede sobrescribir al padre. Usa su **grupo de color propio** `--btn-nav-<estado>-*`
(grupo Button/Nav de Semantic-Color), que se adapta al subtema activo.

## Comportamiento

- **Default**: colores base de navegación.
- **Hover** (`:hover:not(:disabled)`): tokens de hover.
- **Focus** (`:focus-visible`): `outline: none`; el foco visible se aporta vía tokens de color.
- **Selected** (`data-selected="true"`, prop `selected`): ítem activo persistente.
- **Disabled** (`:disabled`, prop `disabled`): colores atenuados y `cursor: not-allowed`.
- Botón no controlado: el estado `selected` lo gobierna quien lo usa.
- El texto no hace wrap (`white-space: nowrap`).

## Tokens

| Token | Uso |
| --- | --- |
| `--btn-nav-<estado>-fill` | Fondo (estado ∈ default/hover/focused/selected/disabled) |
| `--btn-nav-<estado>-text` | Color de texto |
| `--btn-nav-<estado>-stroke` | Color del borde |
| `--btn-nav-<estado>-icon` | Color del icono |
| `--btn-xs-size` | Alto del botón |
| `--btn-xs-padding` | Padding lateral |
| `--sp-2` | Separación entre icono y texto |
| `--font-family-cta`, `--font-weight-cta`, `--letter-spacing-cta` | Tipografía CTA |

## Propiedades

| Prop | Tipo | Default | Descripción |
| --- | --- | --- | --- |
| `selected` | `boolean` | `false` | Ítem activo |
| `disabled` | `boolean` | `false` | Deshabilitado |
| `iconLeft` | `string` | — | Icono a la izquierda |
| `iconRight` | `string` | `CaretDown` | Icono a la derecha (caret desplegable) |
| `children` | `node` | — | Texto |
| `className` | `string` | `''` | Clases adicionales |

## Accesibilidad

- Renderiza un `<button type="button">` nativo, con foco y activación por teclado.
- Estado `disabled` mediante el atributo nativo `disabled`.
- El caret por defecto comunica visualmente que abre un desplegable; cuando se use como disparador de un menú, aporta la semántica ARIA correspondiente (p. ej. `aria-expanded`) desde quien lo usa.
- El foco visible y el contraste se apoyan en los tokens de color adaptados al subtema.
